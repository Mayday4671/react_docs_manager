# 海康API签名问题排查步骤

## 问题现象
调用海康威视API时报签名验证失败错误：
```json
{
  "code": "0x02401003",
  "msg": "api AK/SK signature authentication failed,Invalid Signature!"
}
```

## 排查步骤

### 步骤1: 重启开发服务器（重要！）
代码已经修复，但需要重启服务器才能生效：

```bash
# 停止当前运行的服务器（Ctrl+C）
# 然后重新启动
npm run dev
```

### 步骤2: 检查配置
确保你的配置正确：

1. **baseUrl**: `https://192.168.30.3:443/artemis`
   - 必须包含协议（https://）
   - 必须包含端口（:443）
   - 必须包含路径前缀（/artemis）

2. **AppKey**: 你的实际AppKey（如：20522666）

3. **AppSecret**: 你的实际AppSecret（保密）

### 步骤3: 查看浏览器控制台日志
打开浏览器开发者工具（F12），切换到Console标签，查看日志输出：

应该看到类似这样的日志：
```
路径处理: {
  baseUrl: 'https://192.168.30.3:443/artemis',
  baseUrlPath: '/artemis',
  originalPath: '/api/video/v2/cameras/previewURLs',
  fullPath: '/artemis/api/video/v2/cameras/previewURLs',
  method: 'POST'
}

URL构建: {
  baseOrigin: 'https://192.168.30.3:443',
  fullPath: '/artemis/api/video/v2/cameras/previewURLs',
  finalUrl: 'https://192.168.30.3:443/artemis/api/video/v2/cameras/previewURLs'
}
```

### 步骤4: 检查签名算法日志
在后端日志中应该看到：

```
签名函数输入参数: {
  path: '/artemis/api/video/v2/cameras/previewURLs',
  appKey: '20522666',
  appSecret: 'xxxxx***',
  method: 'POST'
}

待签名字符串:
POST
*/*
application/json
x-ca-key:20522666
/artemis/api/video/v2/cameras/previewURLs

生成的签名: xxxxxxxxxxxxxxxxxx
```

### 步骤5: 验证请求体
确保请求体JSON格式正确，包含必需字段：

```json
{
  "cameraIndexCode": "你的实际点位编码",
  "streamType": 0,
  "protocol": "rtsp",
  "transmode": 1,
  "expand": "transcode=0",
  "streamform": "ps"
}
```

**注意**: `cameraIndexCode` 必须是你系统中实际存在的点位编码，示例值 `748d84750e3a4a5bbad3cd4af9ed5101` 可能在你的系统中不存在。

### 步骤6: 测试其他API
如果"获取点位预览取流URLv2"仍然失败，尝试测试一个更简单的API，比如：

- "获取根区域信息" (`/api/resource/v1/regions/root`) - GET请求，无需请求体
- "查询区域列表" - 可能需要更少的参数

## 常见问题

### Q1: 为什么修改代码后还是报错？
**A**: 必须重启开发服务器！Next.js的API路由不会热重载。

### Q2: 如何确认代码已经更新？
**A**: 查看浏览器控制台日志，如果看到"路径处理"和"URL构建"的日志，说明新代码已生效。

### Q3: 签名算法是否正确？
**A**: 签名算法已经完全按照Java版本的HKSecretUtils实现：
- 签名字符串格式：`METHOD\n*/*\napplication/json\nx-ca-key:APP_KEY\nPATH`
- 使用HMAC-SHA256算法
- Base64编码

### Q4: 路径拼接是否正确？
**A**: 代码会自动处理：
- 如果baseUrl包含路径前缀（如/artemis），会自动补全到API路径前
- 签名和请求使用同一个完整路径
- 支持HTTPS自签名证书

### Q5: 如果还是报签名错误怎么办？
**A**: 请提供以下信息：
1. 浏览器控制台的完整日志（包括"路径处理"和"URL构建"）
2. 后端日志中的"签名函数输入参数"和"待签名字符串"
3. 完整的错误响应
4. 你使用的baseUrl配置
