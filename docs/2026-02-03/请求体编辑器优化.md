# 请求体编辑器优化

## 问题

1. **请求体不是JSON代码块模式** - 已经是可编辑的文本框了，这个没问题
2. **重置示例会丢失参数** - 之前硬编码为 `{"pageNo": 1, "pageSize": 10}`，丢失了API原始示例
3. **没法保存编辑** - 每次都要重新填写，很麻烦

## 解决方案

### 1. 修复重置示例逻辑

**之前**:
```typescript
onClick={() => {
  setEditedRequestBody('{\n  "pageNo": 1,\n  "pageSize": 10\n}');
}}
```

**现在**:
```typescript
onClick={() => {
  // 重置为API的原始示例
  if (selectedApi?.requestBody) {
    try {
      const parsed = JSON.parse(selectedApi.requestBody);
      setEditedRequestBody(JSON.stringify(parsed, null, 2));
      message.success('已重置为原始示例');
    } catch {
      setEditedRequestBody(selectedApi.requestBody);
      message.success('已重置为原始示例');
    }
  } else {
    setEditedRequestBody('{}');
    message.info('该API没有示例请求体');
  }
}}
```

### 2. 添加保存功能

新增"保存"按钮，将编辑后的请求体保存到 localStorage：

```typescript
<Button 
  size="small"
  type="primary"
  onClick={() => {
    if (selectedApi) {
      try {
        // 验证JSON格式
        JSON.parse(editedRequestBody);
        const savedBodies = JSON.parse(localStorage.getItem('hk-api-saved-bodies') || '{}');
        savedBodies[selectedApi.id] = editedRequestBody;
        localStorage.setItem('hk-api-saved-bodies', JSON.stringify(savedBodies));
        setIsUsingSavedBody(true);
        message.success('请求体已保存');
      } catch {
        message.error('JSON格式错误，无法保存');
      }
    }
  }}
>
  保存
</Button>
```

### 3. 自动加载保存的请求体

修改 `handleViewApiDetail` 函数，优先加载保存的请求体：

```typescript
const handleViewApiDetail = (api: HkApi) => {
  setSelectedApi(api);
  setDrawerVisible(true);
  
  // 优先使用保存的请求体，否则使用API的示例
  try {
    const savedBodies = JSON.parse(localStorage.getItem('hk-api-saved-bodies') || '{}');
    if (savedBodies[api.id]) {
      // 使用保存的请求体
      setEditedRequestBody(savedBodies[api.id]);
      setIsUsingSavedBody(true);
    } else if (api.requestBody) {
      // 使用API的示例请求体
      setEditedRequestBody(JSON.stringify(JSON.parse(api.requestBody), null, 2));
      setIsUsingSavedBody(false);
    } else {
      setEditedRequestBody('{}');
      setIsUsingSavedBody(false);
    }
  } catch {
    // 降级处理
    if (api.requestBody) {
      setEditedRequestBody(api.requestBody);
    } else {
      setEditedRequestBody('{}');
    }
    setIsUsingSavedBody(false);
  }
  
  setDebugResponse('');
};
```

### 4. 添加状态提示

在标题旁边显示"已保存"标签：

```typescript
<Space size="small">
  <Text strong>请求体（可编辑）：</Text>
  {isUsingSavedBody && (
    <Tag color="blue" style={{ fontSize: '11px', padding: '0 6px' }}>已保存</Tag>
  )}
</Space>
```

## 使用流程

### 首次使用
1. 点击API，自动加载示例请求体
2. 编辑请求体（修改参数值）
3. 点击"保存"按钮
4. 显示"已保存"标签

### 再次使用
1. 点击同一个API
2. 自动加载之前保存的请求体
3. 显示"已保存"标签
4. 可以继续编辑和保存

### 重置示例
1. 点击"重置示例"按钮
2. 恢复为API的原始示例
3. "已保存"标签消失
4. 可以重新编辑和保存

## 数据存储

使用 localStorage 存储，格式：

```json
{
  "hk-api-saved-bodies": {
    "1": "{\"cameraIndexCode\": \"...\", ...}",
    "2": "{\"pageNo\": 1, ...}",
    ...
  }
}
```

- **键**: API的ID
- **值**: JSON字符串格式的请求体

## 优点

1. **持久化** - 刷新页面后仍然保留
2. **按API保存** - 每个API独立保存，互不影响
3. **可重置** - 随时可以恢复原始示例
4. **有提示** - 清楚知道当前使用的是保存的还是示例

## 注意事项

1. localStorage有大小限制（通常5MB），但对于API请求体来说足够了
2. 清除浏览器数据会丢失保存的请求体
3. 不同浏览器的localStorage是独立的
4. 保存前会验证JSON格式，格式错误无法保存
