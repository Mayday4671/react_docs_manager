# 海康API查询参数排序和编码修复

## 问题分析

从后端日志发现两个关键问题：

### 问题1: URL编码
我们的代码使用 `URLSearchParams` 将 `#userId#` 编码成了 `%23userId%23`，但海康期望的是原始值 `#userId#`。

**原因**: 签名时必须使用原始值，不能进行URL编码。

### 问题2: 参数顺序
- 我们的顺序: `userId=#userId#&domainId=0`
- 海康期望: `domainId=0&userId=#userId#`

**原因**: 海康威视API要求查询参数按**字母顺序**排序。

## 修复方案

### 1. 不进行URL编码
签名时使用原始参数值，不进行URL编码：

```typescript
// 错误做法
const params = new URLSearchParams(config.requestParams);
const queryString = params.toString(); // 会编码 # 为 %23

// 正确做法
const paramPairs: string[] = [];
for (const key of sortedKeys) {
  paramPairs.push(`${key}=${config.requestParams[key]}`); // 使用原始值
}
const queryString = paramPairs.join('&');
```

### 2. 按字母顺序排序参数
```typescript
// 按字母顺序排序参数键
const sortedKeys = Object.keys(config.requestParams).sort();

// 按排序后的顺序构建查询字符串
for (const key of sortedKeys) {
  paramPairs.push(`${key}=${config.requestParams[key]}`);
}
```

### 示例
对于参数 `{ userId: '#userId#', domainId: '0' }`：

1. **排序**: `['domainId', 'userId']` (字母顺序)
2. **构建**: `domainId=0&userId=#userId#`
3. **签名路径**: `/artemis/api/video/v2/cameras/previewURLs?domainId=0&userId=#userId#`

## 关于参数顺序的说明

海康威视API的签名规范要求：
1. 查询参数必须按**字母顺序**排序
2. 参数值使用原始值，不进行URL编码
3. 格式：`key1=value1&key2=value2`

这与标准的HTTP规范不同，是海康API的特殊要求。

## 注意事项

1. **参数顺序**: 数据库中存储的JSON对象顺序不重要，代码会自动按字母顺序排序
2. **URL编码**: 签名时不编码，实际请求时也不编码（保持一致）
3. **特殊字符**: `#` 等特殊字符在签名时保持原样
