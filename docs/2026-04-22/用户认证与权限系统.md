# 用户认证与权限系统

## 完成时间
2026-04-22

## 功能概述
实现完整的用户认证和基于角色的权限控制系统（RBAC），包括登录注册、JWT 认证、
角色菜单权限配置、按钮级权限控制和个人信息管理。

## 主要功能
- ✅ 登录页面（账号密码登录，JWT Token 认证）
- ✅ 注册页面（开放注册，密码强度校验）
- ✅ JWT 认证（存 localStorage，7天有效期）
- ✅ 路由守卫（未登录自动跳转登录页）
- ✅ 权限菜单（登录后侧边栏只显示角色有权限的菜单）
- ✅ 按钮权限（usePermission('user:add') Hook 控制按钮显示）
- ✅ 无角色用户只能看到首页
- ✅ 个人信息页（修改邮箱/手机号、修改密码）
- ✅ Header 右上角用户头像下拉菜单（个人信息/退出登录）
- ✅ 角色菜单权限配置 API（/api/role-menus）
- ✅ 密码 bcrypt 加密（已迁移现有用户密码）

## 权限控制流程
```
登录 → JWT 存 localStorage → 请求 GET /api/auth
→ 返回 { user, menus: [...], perms: ['user:add', ...] }
→ 侧边栏只渲染有权限的菜单
→ 按钮用 usePermission('user:add') hook 控制显示
```

## 数据库变更

## 使用说明

### 配置角色菜单权限
通过 POST /api/role-menus 为角色分配菜单：
```json
{ "roleId": 1, "menuIds": [1, 2, 3, 5] }
```

### 按钮权限使用
在菜单管理中创建 menuType=F 的按钮节点，设置 perms 字段（如 user:add），
然后在组件中使用：
```tsx
const canAdd = usePermission('user:add');
return canAdd ? <Button>新增</Button> : null;
```

## 注意事项
- JWT_SECRET 在 .env 中配置，生产环境必须替换为随机长字符串
- 现有用户密码已迁移为 bcrypt 哈希，原明文密码仍可登录（迁移后密码不变）
- 角色菜单权限需要在角色管理中手动配置，新角色默认无任何菜单权限
