# 菜单管理

## 完成时间
2026-04-22

## 功能概述
新增菜单管理页面，支持通过可视化界面管理数据库中的菜单数据，无需直接操作数据库。
同时扩展了菜单 API，支持完整的增删改查及 Key 唯一性校验。

## 主要功能
- ✅ 树形表格展示所有菜单（含禁用/隐藏），默认全部展开
- ✅ 新增顶级菜单 / 在目录下新增子菜单
- ✅ 编辑菜单（名称、图标、路径、组件绑定、排序等）
- ✅ 删除菜单（有子菜单时禁止删除，前端拦截）
- ✅ 行内快速切换显示/隐藏、启用/禁用（Switch 组件）
- ✅ **Key 自动生成**：填写名称后自动调后端生成不重复 key（中文词映射 + kebab-case）
- ✅ **Key 实时校验**：手动修改 key 时实时查库校验唯一性，重复则提示
- ✅ **可视化图标选择器**：网格展示 50+ 常用图标，支持搜索过滤，选中高亮
- ✅ 菜单名称列内联显示图标，无子菜单行不显示展开按钮
- ✅ 组件绑定下拉选择（列出 componentMap 中已注册的 key）

## Key 生成逻辑
```
用户填名称 → 前端发请求 → 后端生成候选 key
→ 中文词按映射表转英文（如「用户管理」→ user-management）
→ 查库校验唯一性 → 重复则追加 -2/-3 → 返回不重复的 key → 填入表单
```

### 数据库

## 注意事项
- 菜单 key 与 `componentMap.tsx` 中的组件 key 必须一致，否则点击菜单不会渲染对应组件
- 新增组件后需同步更新 `MenuManagement.tsx` 中的 `REGISTERED_KEYS` 常量
- `generateUniqueKey` 的中文映射表目前只覆盖常用词，生僻词会被转为连字符占位
