## 二、文档笔记导出

### 功能概述
在文档查看器顶部新增「导出」下拉按钮，支持多种格式导出。

### 主要功能
- ✅ **下载原文件**：按导入时的真实后缀还原（`.md`、`.sh`、`.bat`、`.txt`、`.docx` 等）
- ✅ **导出 PDF**：将当前渲染内容截图生成 PDF（html2canvas + jsPDF）
- ✅ **转换为 Word**：仅 Markdown 笔记可用，将 md 内容转为 `.docx`（docx 库）
- ✅ 导入时保留真实文件后缀到 `fileType` 字段（原来统一存 `'md'`）
- ✅ 大文件导入限制：文本文件 ≤ 5MB，二进制文件 ≤ 20MB，超限跳过并提示

### 注意事项
- 已存在的旧笔记 `fileType` 仍为 `'md'`，重新导入才会存真实后缀
- PDF 导出基于页面截图，中文字体依赖浏览器渲染

### 相关文件
- `src/frontend/components/docs/DocNoteViewer.tsx` — 新增导出按钮和导出逻辑
- `src/frontend/components/docs/DocNotes.tsx` — 导入时保留真实后缀，加大小限制

### 新增依赖
```
html2canvas   — DOM 截图
jspdf         — PDF 生成
docx          — Word 文档生成
```

## 四、菜单管理

### 功能概述
新增菜单管理页面，支持通过 UI 界面管理数据库中的菜单数据，无需直接操作数据库。

### 主要功能
- ✅ 树形表格展示所有菜单（含禁用/隐藏的）
- ✅ 新增顶级菜单 / 在目录下新增子菜单
- ✅ 编辑菜单（名称、图标、路径、组件、排序等）
- ✅ 删除菜单（有子菜单时禁止删除）
- ✅ 行内快速切换显示/隐藏、启用/禁用
- ✅ **Key 自动生成**：填写名称后自动调后端生成不重复 key
- ✅ **Key 实时校验**：手动修改 key 时实时查库校验唯一性，重复则提示
- ✅ 组件绑定下拉选择（列出 componentMap 中已注册的 key）

### Key 生成逻辑
```
用户填名称 → 前端发请求 → 后端生成候选 key（中文转英文关键词 + kebab-case）
→ 查库校验唯一性 → 重复则追加 -2/-3 → 返回不重复的 key → 填入表单
```

### 相关文件
- `src/frontend/components/system/MenuManagement.tsx` — 菜单管理主组件
- `src/frontend/config/componentMap.tsx` — 注册 `menu-management` 组件
- `src/backend/services/menuService.ts` — 新增 `getAllMenusForAdmin()`、`generateUniqueKey()`、`checkKeyExists()`
- `src/app/api/menu/route.ts` — 扩展支持 POST/PUT/DELETE 及 key 生成/校验接口
- `prisma/add-menu-management.ts` — 插入菜单管理菜单项脚本（已执行）
