# 首页底部卡片扩充计划 (V5)

在精简布局的基础上，利用底部空间增加两个信息展示卡片，提升页面的功能性与丰富度。

## 拟議变更

### 1. 底部卡片布局 (Bottom Card Layout)

- **左右分栏**: 在图表区域下方，新增一排 `Row`。
  - **左侧: 通知公告 (Notice)**: 占据 12/24 或 16/24 宽度。
  - **右侧: 更新日志 (Changelog)**: 占据剩余宽度。
- **一屏适配**: 严格控制列表项数量，确保在无需滚动的情况下完整展示。

### 2. 内容设计 (Content Design)

- **通知公告**:
  - 包含标题、时间、状态标签（如“重要”、“新”）。
  - 使用 `List` 组件实现。
- **更新日志**:
  - 包含版本号、更新日期、核心更新点摘要。
  - 使用 `Timeline` 或精简的 `List` 展示。

### 3. 视觉统一

- 复用 V4 的 `cardStyle` (1px 边框, 16px 圆角)。
- 调优内间距，使内容排布更紧凑。

## 验证计划

### 手动验证

1. **视觉完整性**: 检查底部卡片是否与上方图表对齐，间距是否适中。
2. **滚动条检查**: 确认新增卡片后，首页在主流分辨率下是否依然无需滚动。
3. **响应式**: 检查移动端下卡片是否正确堆叠。
