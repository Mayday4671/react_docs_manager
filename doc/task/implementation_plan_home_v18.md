# 首页图表自适应修复计划 (V18)

彻底解决图表在窗口缩放时无法实时自适应的问题，并进一步优化多端响应式体验。

## 拟議变更

### 1. 强制图表重绘机制

- **状态同步**: 引入一个 `key` 状态，当 `ResizeObserver` 监测到容器尺寸显著变化时，通过改变图表的 `key` 强制 React 重新挂载图表组件，或者通过 `useEffect` 调用图表实例的 `changeSize`。
- **配置解耦**: 确保图表的数据和配置不被重复渲染干扰，而是精准响应容器高度变化。

### 2. 布局权重精调

- **移除硬编码高度**: 彻底移除 `minHeight: 280px` 这种硬骨头，改用 `flex: 1` 配合 `min-height: 0`。
- **视口占比优化**:
  - 大屏下，通过 `vh` 单位为图表行提供保底的基础高度，但允许其根据 Flex 缩放。
  - 修复 `isMobile` 模式下的组件高度失效问题，确保图表在垂直堆叠时有确定的 `px` 高度（如 `260px`）。

### 3. 抗干扰容器

- 为每个图表包裹一层独立的 `position: relative; overflow: hidden` 容器。
- 确保 AntV 的 `autoFit` 监听的是最直接的父容器。

### 4. 彻底清理冗余代码

- 检查并移除 `Home.tsx` 中过去几个版本冲突的 `style` 定义。

## 验证计划

1. **实时缩放测试**: 手动拖拽窗口，观察图表是否实时跟随父容器宽高进行平滑缩放。
2. **多终端模拟**: 切换开发者工具的模拟器，验证平板和手机布局下的图表展示。
3. **数据完整性**: 确认图表在被压缩到最小时，核心刻度线依然存在。
