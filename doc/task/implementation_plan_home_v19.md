# 首页手机与平板端适配重构计划 (V19)

彻底解决移动端模式下布局挤压、内容被遮挡以及视觉比例失调的问题。

## 拟議变更

### 1. 视口与容器重置

- **根级容器锁定**: 确保 `Home` 组件容器拥有 `width: 100%; min-width: 0;`。
- **自适应滚动**:
  - 手机端强制开启 `overflow-y: auto`。
  - 桌面端维持 `overflow: hidden` 一屏全览。

### 2. 快捷入口组件升级 (Mobile First)

- **断点逻辑**:
  - `xs/sm`: 切换为双列并排（`span: 12`），并移除 Desc 背景描述，仅保留图标和标题。
  - `md`: 维持四列并排。
- **Padding 削减**: 手机端卡片内边距从 `16px` 减至 `8px`。

### 3. 图表与列表垂直堆叠

- **强制占满**: 确保手机端下所有的 `Col` 都是 `span: 24`。
- **高度保底**: 图表在手机端强制锁定为 `240px` 高度，保证数据细节不被压缩。

### 4. 侧边栏干扰修复

- 如果侧边栏在手机端仍然占据空间，建议在 Home 组件中检测并给出一个警示或通过 CSS 强制内容区覆盖。

## 验证计划

1. **手机模拟器校验**: 使用 iPhone 14 Pro 尺寸，确保通知项不再被切断，快捷入口清晰可见。
2. **横竖屏切换**: 验证在平板横屏和手机竖屏下的布局自适应。
