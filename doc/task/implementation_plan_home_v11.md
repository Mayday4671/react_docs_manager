# 首页卡片动态高度自适应深度优化 (V11)

彻底解决由于高度计算偏差导致的列表项“截断”问题。

## 拟議变更

### 1. 强化高度感应机制

- **ResizeObserver 监听**: 弃用简单的 `useLayoutEffect` 一次性获取，改为使用 `ResizeObserver` 持续监听卡片 Body 的内容区高度。
- **排除头部间距**: 精准计算 `Card` 标题区、按钮区后的“纯净内容区”高度。

### 2. 精准高度建模

- **Alert 项高度解析**:
  - 标准高度 = 20px (内容) + 20px (上下 Padding) + 8px (Margin-Bottom) = 48px。
  - 考虑到文字溢出可能导致双行，进一步收紧渲染逻辑，采用保守高度 52px。

### 3. 数据驱动的渲染限制

- **自适应切割**: `count = Math.floor((containerHeight - 4) / 52)`。
- **底部最后一条处理**: 增加 `overflow: hidden` 并结合 `margin-top: 0` 处理，确保即使计算有微小偏差，不完整的项也会被物理隐藏，绝不显示一半。

## 验证计划

1. **极端高度测试**: 将卡片高度拉到极小或极大，观察显示的通知条数是否能瞬间对齐且底线平齐。
2. **文本长度测试**: 放入超长文本，确认 `ellipsis` 和高度逻辑是否协同。
