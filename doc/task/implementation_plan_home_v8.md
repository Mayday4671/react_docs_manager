# 首页布局深度精调计划 (V8)

解决用户手动改动后的布局失衡，实现文本不溢出、全屏无滚动的最终状态。

## 拟議变更

### 1. 文本溢出防护

- **通知项**: 确保 `Text` 组件使用 `ellipsis` 属性，并结合 `maxWidth` 或 `flex: 1` 机制，保证超长标题不会撑开容器或导致布局变形。
- **日志项**: 版本描述信息强制单行省略，保持列表整齐划一。

### 2. 高度绝对控制

- **容器锁定**: 为外层 `div` 或卡片内容区强制设定固定的 `px` 高度（如通知卡片 220px，日志卡片 220px）。
- **图表同步**: 保持图表 280px 的精简高度，确保整体纵向占位不超过 100vh。

### 3. 视觉一致性修复

- 恢复被用户手动修改可能导致的格式不整：统一卡片 Padding、标题间距等。

## 验证计划

1. **多案例测试**: 填入极长字符串 Mock 数据，验证是否能完美触发省略号而不破坏布局。
2. **零滚动条校验**: 确保在 1080p 屏幕下右侧没有任何滚动区域。
3. **视觉平衡度**: 检查左右卡片高度是否视觉对齐。
