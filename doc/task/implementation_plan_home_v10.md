# 首页卡片动态高度自适应优化 (V10)

解决固定高度导致的列表项“截断（显示一半）”问题，实现基于容器高度的动态条数过滤。

## 拟議变更

### 1. 引入高度感应机制

- **Ref 绑定**: 为底部卡片的内容区（Container）添加 `useRef`。
- **实时高度获取**: 使用 `useLayoutEffect` 或 `useEffect` 动态读取容器高度。

### 2. 动态条数计算逻辑

- **单项高度标准**: 根据 Alert 样式的 `padding`、`margin` 和文本行高，定义一个标准项高度（如 48px）。
- **计算策略**: `显示数量 = Math.floor(容器有效高度 / 单项标准高度)`。
- **数据截断处理**: 根据计算出的数量对 `notices` 和 `changelog` 数组进行 `slice` 处理，确保渲染的每一项都有足够的空间完整展示。

### 3. 响应式联动

- 确保在浏览器窗口缩放时，能够重新计算条数并平滑更新 UI。

## 验证计划

1. **多高度测试**: 手动在浏览器中改变卡片高度，验证显示的条数是否随之动态增减。
2. **零截断校验**: 观察最后一条显示的内容，确保其底部边框或阴影完整可见，无一半显示现象。
3. **性能评估**: 确保动态计算不会引起频繁的重新渲染或性能抖动。
