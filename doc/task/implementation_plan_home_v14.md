# 首页布局最终稳定版本计划 (V14)

解决 V13 方案导致的图表不显示、容器遮挡及高度失效的灾难性问题。

## 拟議变更

### 1. 布局骨架重写

- **主容器**: 采用 `display: flex; flex-direction: column; overflow: hidden;`。
- **强制约束**: 在所有 `Row` 和 `Col` 上添加 `min-height: 0`。这是 Flex 布局中防止内容撑爆父级的核心开关。

### 2. 空间分配再平衡

- **图表区**: 设置具体的高度百分比（如 `40%`）或带 `min-height` 的 flex 占比。
- **列表区**: 设置 `flex: 1; min-height: 0;`。

### 3. 图表可见性修复

- 确保父级容器有确定的 `px` 或 `%` 高度，图表的 `autoFit` 才能工作。
- 暂时移除过度复杂的 `calcVisibleCount` 逻辑，改为 `高度 100% + overflow: hidden` 以确保首要展示的完整性。

### 4. 彻底移除全局 CSS 污染

- 移除对 `html, body` 的暴力修改，改为在 Home 组件内部通过内联样式锁定视口。

## 验证计划

1. **图表显示检查**: 确保 Area 和 Pie 图表在任何缩放比例下都可见。
2. **截断问题检查**: 确认最后一条通知项不再被切断。
