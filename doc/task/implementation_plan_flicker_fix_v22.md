# 视觉闪烁消除修复计划 (V22)

针对 H5Player 在标签页切换或侧边栏收缩瞬间出现的“画面残留”与“重绘闪烁”问题进行深度优化。

## 拟議变更

### 1. 尺寸监听防抖 (Debounce)

- **优化 H5Player.tsx**: 在 `ResizeObserver` 回调中引入防抖逻辑（如 50-100ms），防止由于布局引擎计算过程中的中间态尺寸触发播放器重绘。

### 2. 精准容器锁定

- **CSS 增强**: 锁定 `#player` 及其父容器的 `background-color` 为纯黑，并确保在 Resize 期间容器高度不会出现瞬间的“塌陷”。

### 3. Resize 时机优化

- **双重触发机制**: 当可见性切换时（从 `display: none` 回归），先等待一次 `requestAnimationFrame` 之后再执行 `JS_Resize`，确保 DOM 已完全就绪。

## 验证计划

1. **高频切换测试**: 快速在“首页”和“视频页”之间切换，验证是否还有明显的黑屏或画面残留。
2. **侧边栏拉伸测试**: 缓慢拖动或快速点击侧边栏收缩，观察视频重绘的平滑度。
