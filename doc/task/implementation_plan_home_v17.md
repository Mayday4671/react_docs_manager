# 首页多端适配增强计划 (V17)

根据用户建议，在窗口缩到一定比例时，自动切换为平板/手机模式，确保组件宽度与可见性。

## 拟議变更

### 1. 响应式布局切换 (Grid + Breakpoints)

- **容器逻辑分流**:
  - **大屏 (lg 以上)**: 继续维持 `height: 100%; overflow: hidden;`，采用动态切片实现 Dashboard 式的一屏全览。
  - **中/小屏 (md 以下)**: 切换为 `height: auto; overflow-y: auto;`。此时主容器将高度撑开，允许页面上下滚动，以自适应平板或手机视口。
- **栅格调整**:
  - 中部图表与底部卡片在 `xs/sm/md` 断点下从 `Row` 的横向排布切换为全宽 (`span: 24`) 纵向堆叠。

### 2. 组件保底高度与显示

- **图表保底**: 为图表容器设置 `min-height: 280px`，确保在移动端垂直堆叠时，图表不会因为失去父级 flex 支撑而坍塌。
- **列表显示**: 在移动端模式下，暂时禁用动态切片（slice），展示全部通知/日志项，或者展示固定数量（如 8 条），让用户通过滚动阅读。

### 3. 主容器 CSS 动态切换

- 使用 Ant Design 的 `Grid.useBreakpoint()` Hook 实时捕捉当前屏幕等级。
- 根据 `breakpoints.md` 等状态动态调整最外层 Div 的 `overflow` 和 `height` 样式。

### 4. 视觉优化

- 移动端下减小卡片之间的 `gutter` (从 12px 减至 8px)，提升紧凑感。
- 调整 Title 和 Text 的大屏 `clamp` 值，确保在窄屏下不发生文字折行。

## 验证计划

1. **断点切换验证**: 将浏览器窗口拉窄至类似手机的比例，观察布局是否自动变为单列堆叠并产生滚动条。
2. **图表完整性**: 确认在移动模式下，Area 图表的 X 轴展示是否依然完整（因保底高度存在）。
3. **回归测试**: 确保在大屏模式下的“一屏全览” Dashboard 感保持不变。
