# 标签页切换数据丢失修复计划 (V20)

解决在系统标签栏切换时，首页或其他页面出现数据重置或展示异常的问题。

## 拟議变更

### 1. 布局策略优化 (LayOut.tsx)

- **排查渲染逻辑**: 检查 `LayOut.tsx` 切换标签页时是物理卸载组件还是仅仅隐藏。
- **引入 Keep-Alive 思想**: 如果业务允许，尝试将 Tab 内容改用 `display: none` 方式隐藏而非完全卸载，以保留组件内部状态（如已加载的数据和滚动位置）。

### 2. 首页状态鲁棒性增强 (Home.tsx)

- **初始值校准**: 为 `visibleCounts` 提供更稳健的默认值，防止在页面恢复挂载初期的瞬间渲染为空白。
- **Key 关联优化**: 检查 `chartKey` 是否在不平衡的时刻被重置。

### 3. 数据层级提升

- 如果发现核心展示数据（如通知、日志）每次都要重新获取，建议将其移入父级 Context 或使用简单的 SessionStorage 缓存。

## 验证计划

1. **切换回归测试**: 在首页加载完毕后，切换到其他页面（如“用户中心”），再切回“首页”，观察图表是否立即显示且列表数量是否恢复正确。
2. **移动端验证**: 确保跨端下的重绘逻辑在 Tab 切换后依然有效。
