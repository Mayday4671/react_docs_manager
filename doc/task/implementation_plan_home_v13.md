# 首页自适应布局终极修复计划 (V13)

解决 V12 版本中由于 Flex 比例失调导致的图表占位过大及内容截断问题。

## 拟議变更

### 1. 结构化视口管理 (VH 方案)

- **主容器**: 弃用 `height: 100%`，改用 `height: 100%` 结合外层容器的 `display: flex` 或 `min-height: 0`。
- **空间再分配**:
  - **快捷入口**: 固定占位，不参与缩放。
  - **中部图表**: 大幅压缩比例，通过 `max-height` 或更小的 `flex-grow` 确保底部卡片能展示 4-6 条数据。

### 2. 移除所有残留的 Hard-coded px

- **Alert 项**: 彻底移除 `height: 42px`。改为使用 `padding` 和 `line-height` 自然撑开。
- **计算逻辑**: `ResizeObserver` 计算条目数时，动态获取渲染后的第一个子项高度，而不是用写死的值。

### 3. 图标比例修正

- 修复图标在大屏或小屏下不协调的问题。
- 确保快捷入口的 Card 布局在响应式下不会产生高度突变。

### 4. 强制物理重绘

- 确保图表在每次窗口大小变化时都能获得父容器的真实高度。

## 验证计划

1. **全分辨率覆盖**: 在 16:9 和 4:3 屏幕下检查底部对齐。
2. **条数动态验证**: 确保增加/减少浏览器高度时，列表项能够干净利落地增减，且不产生“半截”视觉。
