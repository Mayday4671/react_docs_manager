# 首页自适应与图表展示修复 (V16)

彻底解决缩放时图表数据展示不全、布局被遮挡以及列表比例不当的问题。

## 拟議变更

### 1. 布局结构标准化 (AntD 优先)

- **取消硬核视口锁定**: 移除对 `html, body` 的暴力锁定，优先保证 Ant Design 自身容器的流动性。
- **空间占比修正**:
  - 顶部卡片: 固定高度。
  - 中部图表: 设置显著的 `flex-grow` 权重，并通过 `vh` 给予一个稳定的初始高度（如 `35vh`）。
  - 底部卡片: 剩余空间平分。

### 2. 图表渲染优化

- **开启 autoFit**: 确保所有图表参数包含 `autoFit: true`。
- **Padding 调整**: 减小图表的内边距，为轴标签（如 00:00, 04:00 等）在大比例缩放时留出更多像素空间。

### 3. 卡片内容自适应

- **回归经典方案**: 暂时弃用复杂的动态条数算法（因为渲染顺序会导致偏差）。
- **智能截断**: 使用 CSS 的 `overflow: hidden` 配合卡片底部的特定高度，确保即便在缩放时，也不会因为试图显示额外条目而挤压图表。

### 4. 彻底清理冲突代码

- 移除所有 V10-V15 遗留的零散 `position: absolute` 尝试。

## 验证计划

1. **多级缩放测试**: 手动缩放浏览器窗口，观察图表 X 轴标签是否始终可见。
2. **列表遮挡测试**: 确保快捷入口与图表之间、图表与列表之间间距匀称。
3. **环境兼容**: 确保 `npm run dev` 能正常启动且无警告。
